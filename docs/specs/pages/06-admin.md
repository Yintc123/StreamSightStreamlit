# 頁面規格:系統管理

- 頁面編號:6
- 對應模組:模組 5 系統管理
- 存取權限:**僅 Admin**
- 導覽:非 Admin 不註冊此頁面(而非僅隱藏連結)

## 目的

提供 Admin 管理使用者、權限、系統日誌與資料庫狀態,並查詢即時資料的歷史紀錄。

## 版面

以 `st.tabs` 分為四個分頁:使用者 / 權限 / 日誌 / DB 狀態。

## UI 版面規劃

寬版,`st.tabs(["使用者", "權限", "日誌", "DB 狀態"])`。頁面進入前先做 Admin gate。

```
[ 使用者 ] [ 權限 ] [ 日誌 ] [ DB 狀態 ]                 ← st.tabs
── 使用者 ──────────────────────────────────────────────
 搜尋[____] 角色[▾] 狀態[▾]                              ← 篩選列
 帳號  Email  角色  建立時間  狀態   [停用/啟用]          ← st.dataframe/逐列

── 權限 ────────────────────────────────────────────────
 使用者[▾]  角色 user ↔ admin  [ 變更 ]  → st.dialog 二次確認 + 寫日誌

── 日誌 ────────────────────────────────────────────────
 時間範圍  使用者[▾]  等級[▾]  [查詢]
 時間  使用者  動作  結果 (等級著色)   ← st.dataframe

── DB 狀態 ──────────────────────────────────────────────
 ┌─────────┬─────────┬─────────┐
 │ 連線狀態 │ 各表列數 │ DB 大小  │   ← st.columns + st.metric
 └─────────┴─────────┴─────────┘
 即時資料歷史查詢:時間範圍 → st.dataframe
```

| 分頁 | 主要元件 | 備註 |
|---|---|---|
| 使用者 | 篩選 + `st.dataframe` + 停用/啟用 button | 敏感操作寫日誌 |
| 權限 | `st.selectbox` + `st.dialog` 確認 | 不可將自己降為最後一個 Admin |
| 日誌 | 篩選 + `st.dataframe` | 等級以狀態色著色 |
| DB 狀態 | `st.columns` + `st.metric` + 歷史查詢 | 唯讀 |

- 進入頁面即需 Admin(前端不註冊 + 後端再驗);敏感操作全部留稽核。

## 功能細節

### 使用者
- 列出所有使用者(帳號、Email、角色、建立時間、狀態)。
- 支援搜尋與篩選。
- 可停用 / 啟用帳號。

### 權限管理
- 調整使用者角色(`user` ↔ `admin`)。
- 變更需二次確認,並寫入日誌。

### 系統日誌
- 查詢操作 / 系統日誌(時間、使用者、動作、結果)。
- 篩選:時間範圍、使用者、等級。

### DB 狀態監控
- 顯示連線狀態、各表列數、資料庫大小。
- 即時資料歷史查詢:依時間範圍檢視已落地的即時資料。

## 資料

- 透過後端**管理 API** 存取 users / logs / records / realtime;前端不直接連 DB。敏感操作(停用帳號、角色變更)呼叫對應 API,由後端寫稽核日誌。

## 權限規則

- 進入頁面即需 Admin;非 Admin 一律擋下(前端不註冊 + 後端再驗)。
- 角色變更、帳號停用等敏感操作需記錄於日誌。

## 狀態與錯誤處理

進入頁面的存取控制以「前端不註冊 + 後端再驗」為主;頁內操作失敗的呈現一律依 [錯誤處理規格 §3](../error-handling.md#3-呈現契約本規格唯一權威):

| 情境 | 依 §3 呈現 |
|---|---|
| 非 Admin 直接存取 URL | 前端不註冊此頁;若仍打到後端 API → 403 `PermissionDenied` → `st.error`「你沒有權限執行此操作」 |
| 敏感操作被拒(停用帳號 / 角色變更遇 403) | `PermissionDenied` → `st.error`;對明知無權的操作**預先停用按鈕** |
| 不可將自己降級為最後一個 Admin | 後端以 `ValidationError`(對應 422)回絕 → `st.dialog` 內或按鈕旁 `st.error`「無法移除最後一位 Admin」;變更不生效 |
| 查詢 / 載入失敗(使用者、日誌、DB 狀態) | `ApiError` → `st.error` + 保留頁框 + 可重試(附錯誤代碼) |
| 無資料(篩選後) | `st.info` 空狀態,取代對應 `st.dataframe` |

> 敏感操作(停用帳號、角色變更)無論成功或被拒,後端一律寫稽核日誌;前端只負責依上表呈現結果。

## 依賴 / 備註

- 敏感操作建議全部留稽核紀錄。
