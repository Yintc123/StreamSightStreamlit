# 頁面規格:系統管理

- 頁面編號:6
- 對應模組:模組 5 系統管理
- 存取權限:**僅 Admin**
- 導覽:非 Admin 不註冊此頁面(而非僅隱藏連結)

## 目的

提供 Admin 管理使用者、權限、系統日誌與資料庫狀態,並查詢即時資料的歷史紀錄。

## 版面

以 `st.tabs` 分為四個分頁:使用者 / 權限 / 日誌 / DB 狀態。

## 功能細節

### 使用者
- 列出所有使用者(帳號、Email、角色、建立時間、狀態)。
- 支援搜尋與篩選。
- 可停用 / 啟用帳號。

### 權限管理
- 調整使用者角色(`user` ↔ `admin`)。
- 變更需二次確認,並寫入日誌。

### 系統日誌
- 查詢操作 / 系統日誌(時間、使用者、動作、結果)。
- 篩選:時間範圍、使用者、等級。

### DB 狀態監控
- 顯示連線狀態、各表列數、資料庫大小。
- 即時資料歷史查詢:依時間範圍檢視已落地的即時資料。

## 資料

- users 表、logs 表、records / realtime 表(唯讀查詢)。

## 權限規則

- 進入頁面即需 Admin;非 Admin 一律擋下(前端不註冊 + 後端再驗)。
- 角色變更、帳號停用等敏感操作需記錄於日誌。

## 狀態與錯誤處理

- 非 Admin 直接存取 URL → 後端拒絕。
- 不可將自己降級為最後一個 Admin(避免鎖死)。

## 依賴 / 備註

- 敏感操作建議全部留稽核紀錄。
