# 頁面規格:資料管理

- 頁面編號:3
- 對應模組:模組 2 資料管理
- 存取權限:已登入(編輯/刪除限**創建者或 Admin**)
- 導覽:一般使用者可見

## 目的

提供資料的建立、讀取、更新、刪除與批量匯入,並以權限控制編輯與刪除。

## 版面

以 `st.tabs` 分為三個分頁:列表 / 新增 / 匯入。

## UI 版面規劃

寬版,`st.tabs(["列表", "新增", "匯入"])`,三分頁各自版面:

```
[ 列表 ] [ 新增 ] [ 匯入 ]                              ← st.tabs
── 列表 ────────────────────────────────────────────────
 分類[▾]  時間範圍[起]-[迄]  關鍵字[____]  [ 篩選 ]      ← st.columns 篩選列
 ┌──────────────────────────────────────────────────┐
 │ 標題    數值   分類   創建者  時間   [編輯][刪除]   │  ← st.dataframe / 逐列
 │ ...                                               │     動作欄依權限停用
 └──────────────────────────────────────────────────┘
 ‹ 上一頁   第 3 / 20 頁   下一頁 ›                     ← 分頁列 (LIMIT/OFFSET)

── 新增 ────────────────────────────────────────────────
 標題[__________]  數值[____]  分類[▾]   [ 送出 ]        ← st.form

── 匯入 ────────────────────────────────────────────────
 [ 拖曳或選擇 CSV / JSON ]                              ← st.file_uploader
 預覽:st.dataframe(前 N 列) + 欄位對應 + 錯誤列標示
 [ 確認匯入 ]
```

| 分頁 | 主要元件 | 備註 |
|---|---|---|
| 列表 | 篩選 `st.columns` + `st.dataframe` + 分頁 buttons | 編輯/刪除:非創建者且非 Admin → `disabled=True` |
| 編輯 | `st.dialog` 或展開區的 `st.form` | 載入既有值後修改 |
| 刪除 | `st.dialog` 二次確認 | 建議軟刪除 |
| 新增 | `st.form` + 送出 | 自動帶入創建者/timestamp |
| 匯入 | `st.file_uploader` + 預覽 `st.dataframe` | 錯誤列以 danger 色標示,不中斷其餘 |

- 權限判斷(創建者/Admin)在 Python 端決定按鈕 `disabled`;無權限一律停用而非隱藏。

## 功能細節

### 列表(讀取)
- `st.dataframe` 顯示資料,支援欄位排序。
- 篩選:分類、時間範圍、關鍵字(標題)。
- 分頁:每頁 N 筆,提供頁碼控制(SQL `LIMIT/OFFSET`)。
- 每列提供「編輯」「刪除」動作;非創建者且非 Admin 時停用。

### 新增(建立)
- 欄位:標題(text)、數值(number)、分類(select)。
- 自動帶入:創建者、timestamp。
- 提交後寫入資料庫,成功提示並刷新列表。

### 編輯(更新)
- 權限:創建者或 Admin。
- 載入既有值 → 表單修改 → 更新。
- 記錄更新時間(可選)。

### 刪除
- 權限:創建者或 Admin。
- 需二次確認。
- 軟刪除或硬刪除待定(建議軟刪除保留稽核)。

### 匯入(批量)
- `st.file_uploader` 接受 CSV / JSON。
- 以 pandas 解析,顯示預覽與欄位對應。
- 驗證必填/型別,列出錯誤列;通過後批量寫入。

## 資料

- records 表:`id, title, value, category, created_by, created_at, updated_at`。

## 權限規則

| 動作 | 允許者 |
|---|---|
| 讀取 | 所有登入者 |
| 建立 | 所有登入者 |
| 更新 / 刪除 | 該筆創建者、Admin |

## 狀態與錯誤處理

- 匯入格式錯誤 → 標示問題列,不中斷其餘。
- 無權限操作 → 停用按鈕並提示。

## 依賴 / 備註

- 大量資料建議以 SQL 端分頁/篩選,避免全載入前端。
