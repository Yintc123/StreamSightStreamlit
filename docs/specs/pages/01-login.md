# 頁面規格:登入 / 註冊

- 頁面編號:1
- 對應模組:模組 1 認證
- 存取權限:未登入使用者
- 導覽:未登入時為唯一註冊的頁面

## 目的

提供使用者登入與註冊入口,並在成功後建立 session 與角色,決定後續可存取的頁面。

## 版面

以 `st.tabs` 分為兩個分頁:

| 分頁 | 內容 |
|---|---|
| 登入 | 帳號、密碼、登入按鈕 |
| 註冊 | 帳號、密碼、確認密碼、Email、註冊按鈕 |

## 功能細節

### 登入
- 欄位:帳號(text)、密碼(password)。
- 驗證:查使用者表,密碼比對(雜湊)。
- 成功:寫入 `session_state`(`authenticated`、`username`、`role`),導向儀表板。
- 失敗:顯示「帳號或密碼錯誤」,不透露是帳號或密碼哪個錯。

### 註冊
- 欄位:帳號、密碼、確認密碼、Email。
- 驗證:帳號唯一、兩次密碼一致、密碼強度、Email 格式。
- 成功:建立使用者(預設角色 `user`),提示改至登入分頁。
- 失敗:對應欄位顯示錯誤。

## 資料

- users 表:`id, username(唯一), password_hash, email, role, created_at`。
- 角色:`user` / `admin`。

## 狀態與錯誤處理

- 已登入者進入本頁 → 直接導向儀表板。
- 帳號重複、密碼不一致、格式錯誤 → inline 錯誤訊息。

## 依賴 / 備註

- 認證方案:`streamlit-authenticator` 或原生 `st.login`(OIDC),待 [ADR](../../decisions/) 決定。
- 密碼必須雜湊儲存(如 bcrypt),不得明文。
